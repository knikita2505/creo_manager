# Скрипты запуска

Проект включает удобные скрипты для запуска серверов.

## Скрипты

### `start_backend.sh` - Запуск Backend

Автоматически:
- ✅ Создает виртуальное окружение (если не существует)
- ✅ Активирует виртуальное окружение
- ✅ Проверяет наличие `.env` файла
- ✅ Устанавливает зависимости (если нужно)
- ✅ Проверяет наличие FFmpeg
- ✅ Запускает сервер FastAPI

**Использование:**
```bash
./start_backend.sh
```

### `start_frontend.sh` - Запуск Frontend

Автоматически:
- ✅ Проверяет наличие Node.js (>= 18)
- ✅ Устанавливает зависимости (если нужно)
- ✅ Проверяет наличие `.env.local` (опционально)
- ✅ Запускает Next.js dev сервер

**Использование:**
```bash
./start_frontend.sh
```

### `start.sh` - Запуск всего проекта

Запускает backend и frontend в отдельных окнах терминала.

**Использование:**
```bash
./start.sh
```

**Примечание:** 
- На macOS использует встроенный Terminal
- На Linux использует gnome-terminal или xterm
- Для остановки серверов закройте соответствующие окна терминала

## Ручной запуск

Если вы предпочитаете запускать вручную:

### Backend
```bash
cd backend
source venv/bin/activate
python run.py
```

### Frontend
```bash
cd frontend
npm install  # только при первом запуске
npm run dev
```

## Troubleshooting

### Скрипт не запускается

Убедитесь, что скрипт имеет права на выполнение:
```bash
chmod +x start_backend.sh start_frontend.sh start.sh
```

### Ошибка "Permission denied"

Проверьте права доступа:
```bash
ls -la *.sh
```

Должно быть `-rwxr-xr-x`. Если нет - выполните:
```bash
chmod +x *.sh
```

### Ошибка "No such file or directory"

Убедитесь, что вы запускаете скрипт из корневой директории проекта:
```bash
pwd  # должен показать путь к creo_manager
```
