# Создание файла .env

Файл `.env` необходим для работы приложения, но он не включается в git (по соображениям безопасности).

## Создание файла

### Вариант 1: Через терминал (рекомендуется)

```bash
cd backend
cp .env.example .env
```

### Вариант 2: Вручную

1. Скопируйте файл `backend/.env.example`
2. Переименуйте его в `backend/.env`
3. Откройте в редакторе и заполните необходимые значения

## Заполнение значений

### Обязательные параметры для начала работы:

1. **DATABASE_URL** - строка подключения к Supabase
   - Получите в Supabase Dashboard → Settings → Database → Connection string → URI
   - Формат: `postgresql+asyncpg://postgres:[PASSWORD]@db.[REF].supabase.co:5432/postgres`
   - Замените `[PASSWORD]` на ваш пароль БД
   - Замените `[REF]` на идентификатор вашего проекта

2. **SECRET_KEY** - секретный ключ для JWT токенов
   - Сгенерируйте случайную строку (минимум 32 символа)
   - Можно использовать: `python -c "import secrets; print(secrets.token_urlsafe(32))"`

### Опциональные параметры (можно оставить значения по умолчанию):

- Остальные параметры (YouTube, Google Drive, Google Ads, Telegram) можно настроить позже

## Проверка

После создания файла проверьте подключение:

```bash
python scripts/check_db.py
```

## Важно

⚠️ **Никогда не коммитьте файл `.env` в git!**  
Он уже добавлен в `.gitignore` и будет проигнорирован.
